List current user's cron jobs
عرض مهام كرون للمستخدم الحالي
crontab -r

مراقبة سجلات تنفيذ كرون

البحث عن إدخالات كرون في سجل النظام

⚙️ Advanced Usage
@reboot /path/script.sh
Run script at system startup
تشغيل السكريبت عند بدء تشغيل النظام
@yearly /path/backup.sh
Run annually (same as 0 0 1 1 *)
تشغيل سنوياً (مثل 0 0 1 1 *)
@monthly /path/cleanup.sh
Run monthly (same as 0 0 1 * *)
تشغيل شهرياً (مثل 0 0 1 * *)
@weekly /path/report.sh
Run weekly (same as 0 0 * * 0)
تشغيل أسبوعياً (مثل 0 0 * * 0)
💡 Tips & Best Practices
نصائح وأفضل الممارسات
🔒 Security First
Always use absolute paths in cron jobs. The PATH environment variable may be limited in cron context.
استخدم دائماً المسارات المطلقة في مهام كرون. متغير PATH قد يكون محدوداً في بيئة كرون.
📊 Logging is Key
Redirect output to log files:
command >> /var/log/my-job.log 2>&1
توجيه المخرجات إلى ملفات السجل:
command >> /var/log/my-job.log 2>&1
🚫 Avoid Overlapping
Use file locks or PID files to prevent multiple instances of the same job running simultaneously.
استخدم أقفال الملفات أو ملفات PID لمنع تشغيل عدة نسخ من نفس المهمة في نفس الوقت.
🧪 Test First
Test your scripts manually before adding them to cron. Verify permissions and dependencies.
اختبر السكريبتات يدوياً قبل إضافتها لكرون. تحقق من الصلاحيات والتبعيات.
📧 Email Notifications
Set MAILTO variable at the top of crontab to receive job output via email.
تعيين متغير MAILTO في أعلى جدول كرون لاستلام مخرجات المهام عبر البريد الإلكتروني.
⏰ Timezone Awareness
Cron uses system timezone. Set TZ variable if needed:
TZ=UTC 0 0 * * * command
كرون يستخدم المنطقة الزمنية للنظام. تعيين متغير TZ إذا لزم الأمر:
TZ=UTC 0 0 * * * command
